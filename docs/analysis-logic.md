# FX-test 分析ロジック まとめ

## 全体フロー

```
①スウィングポイント検出（各時間軸）
        ↓
②ダウ理論でトレンド判定（各時間軸）
        ↓
③マルチタイム一致チェック
        ↓
④エントリーシグナル算出
        ↓
⑤通知判定
```

---

## ① スウィングポイント検出（`swingPoints.ts`）

### 何をやっているか
ローソク足を時系列順に並べ、着目した足の**左右 lookback 本**と比較して局所的な高値・安値を探す。

### スウィングハイの条件
```
着目足の高値 > 左3本の高値 すべて
かつ
着目足の高値 > 右3本の高値 すべて
```

### スウィングローの条件
```
着目足の安値 < 左3本の安値 すべて
かつ
着目足の安値 < 右3本の安値 すべて
```

### パラメータ
| 変数 | 値 | 意味 |
|------|----|------|
| `lookback` | 3 | 左右何本と比較するか |
| 取得本数 | 直近3個 | 高値・安値それぞれ直近3点を使用 |

### 注意点
- **右 lookback 本が確定していないと検出されない** → 最新足付近のスウィングは拾えない（リアルタイム性の限界）
- 現状 lookback=3 固定。値を増やすと大きな波を、減らすと細かい波を拾う

## ② ダウ理論トレンド判定（`dowTheory.ts`）

### ダウ理論の定義
| 判定 | 条件 |
|------|------|
| **UP（上昇トレンド）** | 高値切り上げ AND 安値切り上げ |
| **DOWN（下降トレンド）** | 高値切り下げ AND 安値切り下げ |
| **RANGE（レンジ）** | それ以外 |

### 実装上の判定ロジック（直近3点を使用）

**高値切り上げ（highsRising）**
```
highs[0] > highs[1]  （最新 > ひとつ前）
かつ
highs[1] > highs[2]  （ひとつ前 > さらにひとつ前）※3点ある場合のみ
```

**安値切り上げ（lowsRising）**
```
lows[0] > lows[1]
かつ
lows[1] > lows[2]  ※3点ある場合のみ
```

→ `highsRising && lowsRising` → **UP**
→ `highsFalling && lowsFalling` → **DOWN**
→ それ以外 → **RANGE**

### 弱点・現状の課題
- スウィングポイントが2点未満しか取れない場合は強制的に **RANGE** 扱い
- 3点目がなくても2点で判定される（厳密さに欠ける）
- 高値と安値の条件が「どちらか一方だけ切り上げ」の場合も RANGE → **トレンド転換の検知が遅い**

## ③ マルチタイム一致チェック（`isTrendAligned`）

### 現状の仕様
```
D1(日足) と H4(4時間足) が一致 → aligned=true、方向決定
H1(1時間足) は一致チェックに含まれていない（エントリー計算に使うだけ）
```

| D1 | H4 | H1 | 結果 |
|----|----|----|------|
| UP | UP | 任意 | ✅ 買い方向で進む |
| DOWN | DOWN | 任意 | ✅ 売り方向で進む |
| その他 | - | - | ❌ エントリー条件未達 |

### 現状の課題
- **H1 がトレンド逆行中でもスルー**される（D1+H4 一致なら通過）
- 将来 M15 を追加しても現状のコードでは参照されない（別途 `isTrendAligned` の修正が必要）

## ④ エントリーシグナル算出（`entrySignal.ts`）

### 買いシグナル（UP方向）
```
エントリー価格 = H1 直近スウィングロー + 5pips  （押し目確認）
SL           = H4 直近スウィングロー - 5pips
SL幅(pips)  = エントリー価格 - SL
TP幅(pips)  = SL幅 × 2  （R/R = 1:2）
TP価格       = エントリー価格 + TP幅
```

### 売りシグナル（DOWN方向）
```
エントリー価格 = H1 直近スウィングハイ - 5pips  （戻り目確認）
SL           = H4 直近スウィングハイ + 5pips
SL幅(pips)  = SL - エントリー価格
TP幅(pips)  = SL幅 × 2
TP価格       = エントリー価格 - TP幅
```

### パラメータ
| 定数 | 値 | 意味 |
|------|----|------|
| `PIP_VALUE` | 0.01 | USDJPY の 1pip |
| `SL_BUFFER_PIPS` | 5 | スウィングポイントからのバッファ |
| `RR_RATIO` | 2 | リスクリワード比 |
| `ENTRY_PROXIMITY_PIPS` | 10 | 通知する距離（現在値からエントリーまで）|

---

## ⑤ 通知判定

```
条件A: isTrendAligned → D1 + H4 が同方向
         ❌ 未達 → 処理終了、通知なし

条件B: proximityPips <= 10  （エントリーまで10pips以内）
         ❌ 未達 → ログのみ、通知なし
         ✅ → ntfy.sh に通知

FORCE_NOTIFY=true の場合は条件B を無視して強制通知
```

---

## 現状の課題まとめ

| # | 課題 | 影響 |
|---|------|------|
| 1 | H1 がトレンド一致チェックに含まれない | D1↑ H4↑ H1↓ でもエントリー計算が走る |
| 2 | M15 未対応 | 短期のエントリータイミング精度が低い |
| 3 | 条件未達でも通知がゼロ | 現状確認のために手動実行が必要 |
| 4 | lookback=3 固定 | 時間軸によって最適値が異なる可能性あり |
| 5 | スウィング右端の確定待ち | 最新 lookback 本のシグナルは常に取れない |
